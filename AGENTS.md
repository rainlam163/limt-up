# LimitUp - 涨停策略选股系统

## 项目概述

涨停策略选股系统是一个量化选股工具，核心目标是在收盘前选出第二天最可能涨停或大涨的股票，输出 Top10 股票列表。

**设计理念**：侧重"趋势延续"和"走势稳定性"，降低盘中波动敏感度，使 14:48 推送结果与收盘后结果保持一致性。

## 技术栈

- **运行时**: Node.js (ES Modules)
- **依赖**: axios (HTTP请求), node-cron (定时任务)
- **数据源**: 腾讯财经API (qt.gtimg.cn)
- **推送服务**: PushPlus (微信公众号推送)

## 核心流程

```
获取市场数据 → 计算市场情绪 → 基础筛选 → 稳定性筛选 → 获取K线数据 → 多因子评分 → 龙头识别 → 诱多识别 → 排序 → 分数阈值过滤 → 输出Top10 → 微信推送
```

## 筛选条件

### 基础筛选

| 条件 | 规则 |
|------|------|
| 市场板块 | 仅主板（600/601/603/605/000/001开头） |
| ST股票 | 排除 |
| 涨幅区间 | 4% ~ 8%（收窄以提高稳定性） |
| 价格区间 | 3 ~ 40 元 |
| 换手率 | > 2% |

### 稳定性筛选

| 条件 | 规则 |
|------|------|
| 振幅阈值 | < 12%（超过可能不稳定） |
| 回撤阈值 | < 5%（最高价与当前价差距） |

### 分数阈值

| 条件 | 规则 |
|------|------|
| 最低分数 | >= 4 分（低于此分数不推送） |

## 多因子评分模型

### 因子权重

| 因子 | 权重 | 说明 |
|------|------|------|
| 趋势延续 | 30% | 连续上涨天数、累计涨幅、均线排列 |
| 量价健康 | 25% | 量比、量价配合度 |
| 走势稳定性 | 10% | 振幅、回撤、价格位置 |
| 动量因子 | 10% | 今日涨幅 |
| 换手率因子 | 5% | 资金博弈程度 |
| 资金因子 | 10% | 成交额规模 |
| 板块热度 | 10% | 市场关注度 |

### 评分规则

**趋势延续因子（最高6分）**
- 连续上涨天数: 1天+1分, 2天+2分, 3天以上+3分
- 3日累计涨幅: 5%-15%+2分, 15%-20%+1分, >20%-1分
- 价格高于5日均价: +1分

**量价健康因子（最高5分）**
- 缩量上涨（量比<1）: +3分
- 温和放量（量比1-1.5）: +2分
- 巨量+大涨（量比>3且涨幅>7%）: -1分
- 连续缩量上涨: +2分

**走势稳定性因子（最高5分，最低-2分）**
- 振幅 ≤ 5%: +2分
- 振幅 5%-7%: +1分
- 振幅 > 10%: -1分
- 回撤 < 1%: +2分
- 回撤 < 2%: +1分
- 回撤 > 4%: -1分
- 价格位置在高位区间(>80%): +1分
- 价格位置在低位区间(<30%): -1分

**诱多识别扣分（最高-3分）**
- 冲高回落: -1分
- 涨停板出货: -1.5分
- 高开回落: -1分
- 低开急拉: -0.5分
- 放量滞涨: -0.5分
- 振幅过大(>10%): -0.5分

**动量因子（最高7分）**
- 涨幅 > 2%: +2分
- 涨幅 > 4%: +2分
- 涨幅 > 6%: +3分
- 连续微调: 涨幅精确值 × 0.1

**换手率因子（最高6.2分）**
- < 3%: 1分
- 3% ~ 6%: 3分
- 6% ~ 10%: 5分
- > 10%: 6分

**资金因子（最高6.2分）**
- > 100亿: 6分
- > 50亿: 4分
- > 10亿: 2分
- 其他: 1分

**板块热度（最高5.1分）**
- 涨幅 > 5%: 5分
- 涨幅 > 3%: 3分
- 其他: 1分

**市场情绪加成（-1 ~ 2分）**
- 极好（涨停>=100）: +2分
- 偏多（涨停>=50）: +1分
- 平稳: +0.5分
- 偏空: 0分
- 极差（跌停>=50）: -1分

## 龙头识别

按涨幅区间划分虚拟板块（涨停板/强势股/活跃股/上涨股），每个板块按成交额排序，取前2名作为龙头，额外加2分。

## 排序规则

多级排序：主分数 → 连续上涨天数 → 涨幅 → 成交额

## 配置说明

`config.js` 主要配置项：

```javascript

PUSHPLUS_TOPIC: ""    // 推送群组
ENABLE_PUSH: true     // 是否启用推送
```

## 定时任务

使用 node-cron 实现定时调度：
- Cron表达式: `00 48 14 * * 1-5`
- 时区: Asia/Shanghai
- 执行时间: 周一至周五 14:48

## 微信推送

使用 PushPlus 服务推送结果到微信：
- 支持白天/夜间模式自动适配
- 卡片式排版，Top3 高亮显示
- 市场情绪卡片（涨停/跌停家数、涨跌比）
- 股票标签：连涨、稳、缩量、热
- 诱多信号警告

**标签说明**
- 🟢 **N连涨**: 连续上涨 ≥ 3天
- 🟡 **稳**: 振幅 < 6%
- 🔵 **缩量**: 量比 < 1.5
- 🔴 **热**: 涨幅 ≥ 7%

获取Token：关注微信公众号"PushPlus推送"，回复"token"获取。

## 开发规范

1. **模块化设计**: 每个因子独立文件，便于维护和调整
2. **ES Modules**: 使用 import/export 语法
3. **异步处理**: 使用 async/await
4. **编码处理**: 腾讯API返回GBK编码，需用 TextDecoder 转换
5. **请求控制**: 批量请求间添加延迟，避免触发限流
6. **浮点精度**: 评分计算使用 toFixed(2) 避免精度问题

## 注意事项

1. 数据来源于腾讯财经API，仅供学习研究
2. 选股结果仅供参考，不构成投资建议
3. 定时任务建议在交易时间内执行（14:48左右）
4. 涨幅区间收窄至 4%-8%，提高 14:48 与收盘结果一致性
5. 稳定性筛选排除振幅过大、回撤过大的股票
6. 诱多识别会扣分，冲高回落、高开回落等形态会被标记
