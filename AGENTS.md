# LimitUp - 涨停策略选股系统

## 项目概述

涨停策略选股系统是一个量化选股工具，核心目标是在收盘前选出第二天最可能涨停或大涨的股票，输出Top10股票列表。

## 技术栈

- **运行时**: Node.js (ES Modules)
- **依赖**: axios (HTTP请求)
- **数据源**: 腾讯财经API (qt.gtimg.cn)
- **推送服务**: PushPlus (微信公众号推送)

## 项目结构

```
limitup/
├── index.js           # 主入口文件
├── config.js          # 配置文件
├── scheduler.js       # 定时任务调度器
├── services/
│   └── market.js      # 市场数据获取服务
├── factors/
│   ├── momentum.js    # 动量因子（涨幅）
│   ├── turnover.js    # 换手率因子
│   ├── capitalFlow.js # 资金流因子（成交额）
│   └── sectorHot.js   # 板块热度因子
├── strategy/
│   └── leader.js      # 龙头识别策略
├── utils/
│   ├── score.js       # 综合评分计算
│   └── push.js        # PushPlus微信推送
└── output/
    └── result.json    # 输出结果
```

## 核心流程

```
获取市场数据 → 股票池过滤 → 多因子评分 → 龙头识别 → 龙头加分 → 排序 → 输出Top10 → 微信推送
```

## 运行命令

```bash
# 立即执行一次
node index.js

# 启动定时任务（周一至周五 14:48 执行）
node index.js schedule

# 后台运行（推荐使用 pm2）
pm2 start index.js --name limitup -- schedule
```

## 筛选条件

| 条件 | 规则 |
|------|------|
| 市场板块 | 仅主板（600/601/603/605/000/001开头） |
| ST股票 | 排除 |
| 涨幅区间 | 3% ~ 9.5%（排除涨停） |
| 价格区间 | 3 ~ 40 元 |
| 换手率 | > 2% |

## 多因子评分模型

### 因子权重

| 因子 | 权重 | 说明 |
|------|------|------|
| 动量因子 | 35% | 今日涨幅 |
| 换手率因子 | 25% | 资金博弈程度 |
| 资金因子 | 25% | 成交额规模 |
| 板块热度 | 15% | 市场关注度 |

### 评分规则

**动量因子（最高7分）**
- 涨幅 > 2%: +2分
- 涨幅 > 4%: +2分
- 涨幅 > 6%: +3分
- 连续微调: 涨幅精确值 × 0.1

**换手率因子（最高6分）**
- < 3%: 1分
- 3% ~ 6%: 3分
- 6% ~ 10%: 5分
- > 10%: 6分

**资金因子（最高6分）**
- > 100亿: 6分
- > 50亿: 4分
- > 10亿: 2分
- 其他: 1分

**板块热度（最高5分）**
- 涨幅 > 5%: 5分
- 涨幅 > 3%: 3分
- 其他: 1分

## 龙头识别

按涨幅区间划分虚拟板块（涨停板/强势股/活跃股/上涨股），每个板块按成交额排序，取前2名作为龙头，额外加2分。

## 排序规则

多级排序：主分数 → 涨幅 → 成交额 → 换手率

## 配置说明

`config.js` 主要配置项：

```javascript
TOP_N: 10           // 输出股票数量
MIN_PRICE: 3        // 最低价格
MAX_PRICE: 40       // 最高价格
MIN_TURNOVER: 2     // 最低换手率
LEADER_BONUS: 2     // 龙头加分
PUSHPLUS_TOKEN: ""  // PushPlus推送Token
ENABLE_PUSH: true   // 是否启用推送
```

## 微信推送

使用 PushPlus 服务推送结果到微信，支持白天/夜间模式自动适配。

获取Token：关注微信公众号"PushPlus推送"，回复"token"获取。

## 开发规范

1. **模块化设计**: 每个因子独立文件，便于维护和调整
2. **ES Modules**: 使用 import/export 语法
3. **异步处理**: 使用 async/await
4. **编码处理**: 腾讯API返回GBK编码，需用 TextDecoder 转换
5. **请求控制**: 批量请求间添加延迟，避免触发限流

## 注意事项

1. 数据来源于腾讯财经API，仅供学习研究
2. 选股结果仅供参考，不构成投资建议
3. 定时任务建议在交易时间内执行（14:48左右）
