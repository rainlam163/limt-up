# LimitUp - 涨停策略选股系统

## 项目概述

涨停策略选股系统是一个量化选股工具，核心目标是在收盘前选出第二天最可能涨停或大涨的股票，输出 Top10 股票列表。

**设计理念**：侧重"趋势延续"和"走势稳定性"，降低盘中波动敏感度，使 14:48 推送结果与收盘后结果保持一致性。

## 技术栈

- **运行时**: Node.js (ES Modules)
- **依赖**: axios (HTTP请求), node-cron (定时任务)
- **数据源**: 腾讯财经API (qt.gtimg.cn)
- **板块数据**: 东方财富API（主）/ 同花顺API（降级）
- **推送服务**: PushPlus (微信公众号推送)

## 核心流程

```
获取市场数据 → 计算市场情绪 → 动态涨幅区间 → 基础筛选 → 稳定性筛选 → 换手率风险过滤 → 获取K线数据 → 获取板块数据 → 动态权重计算 → 多因子评分 → 诱多识别 → 排序 → 分数阈值过滤 → 输出Top10 → 微信推送
```

## 筛选条件

### 基础筛选

| 条件 | 规则 |
|------|------|
| 市场板块 | 仅主板（600/601/603/605/000/001开头） |
| ST股票 | 排除 |
| 涨幅区间 | 4% ~ 8%（动态调整：情绪极好时放宽至9.5%，偏空时收窄至7%） |
| 价格区间 | 3 ~ 40 元 |
| 换手率 | > 2% |

### 换手率风险过滤

| 条件 | 规则 |
|------|------|
| 换手率 > 25% | 直接排除（可能出货） |
| 换手率 > 20% 且 涨幅 > 5% | 排除（警惕出货） |

### 稳定性筛选

| 条件 | 规则 |
|------|------|
| 振幅阈值 | < 12%（情绪差时更严格：< 10%） |

### 分数阈值

| 条件 | 规则 |
|------|------|
| 最低分数 | >= 2.0 分（动态权重体系调整） |

## 动态权重体系

### 基础权重

| 因子 | 基础权重 | 说明 |
|------|----------|------|
| 趋势延续 | 30% | 连续上涨天数、累计涨幅、均线排列 |
| 量价健康 | 25% | 量比、量价配合度 |
| 走势稳定性 | 10% | 振幅、回撤、价格位置 |
| 动量因子 | 10% | 今日涨幅 |
| 资金因子 | 10% | 成交额规模 |
| 板块热度 | 10% | 市场关注度 |
| 换手率 | 5% | 资金博弈程度 |

### 情绪调整系数

根据市场情绪动态调整权重和涨幅区间：

| 市场情绪 | 动量调整 | 稳定性调整 | 涨幅区间 |
|----------|----------|------------|----------|
| 极好（涨停≥100） | ×1.5 | ×0.5 | 4% ~ 9.5% |
| 偏多（涨停≥50） | ×1.2 | ×0.8 | 4% ~ 8% |
| 平稳 | ×1.0 | ×1.0 | 4% ~ 8% |
| 偏空 | ×0.5 | ×1.5 | 4% ~ 7% |
| 极差（跌停≥50） | ×0.3 | ×2.0 | 4% ~ 6% |

## 多因子评分规则

### 趋势延续因子（最高6分）
- 连续上涨天数: 1天+1分, 2天+2分, 3天以上+3分
- 3日累计涨幅: 5%-15%+2分, 15%-20%+1分, >20%-1分
- 价格高于5日均价: +1分

### 量价健康因子（最高5分）
- 缩量上涨（量比<1）: +3分
- 温和放量（量比1-1.5）: +2分
- 巨量+大涨（量比>3且涨幅>7%）: -1分
- 连续缩量上涨: +2分

### 走势稳定性因子（最高5分，最低-2分）
- 振幅 ≤ 5%: +2分
- 振幅 5%-7%: +1分
- 振幅 > 10%: -1分
- 回撤 < 1%: +2分
- 回撤 < 2%: +1分
- 回撤 > 4%: -1分

### 诱多识别扣分（最高-3分）
- 冲高回落: -1分
- 涨停板出货: -1.5分
- 高开回落: -1分
- 低开急拉: -0.5分
- 放量滞涨: -0.5分

### 动量因子（最高7分）
- 涨幅 > 2%: +2分
- 涨幅 > 4%: +2分
- 涨幅 > 6%: +3分

### 换手率因子（最高6分）
- < 3%: 1分
- 3% ~ 6%: 3分
- 6% ~ 10%: 5分
- > 10%: 6分

### 资金因子（最高6分）
- > 100亿: 6分
- > 50亿: 4分
- > 10亿: 2分
- 其他: 1分

### 板块热度（使用热门板块平均涨幅）
- 涨幅 > 5%: 5分
- 涨幅 > 3%: 3分
- 其他: 1分

### 市场情绪加成（-1 ~ 2分）
- 极好（涨停≥100）: +2分
- 偏多（涨停≥50）: +1分
- 平稳: +0.5分
- 偏空: 0分
- 极差（跌停≥50）: -1分

### 换手率风险扣分
- 换手率 > 25%: -1.5分
- 换手率 > 20%: -1分
- 换手率 > 15% 且 涨幅 > 6%: -0.5分

## 排序规则

多级排序：主分数 → 板块热度 → 连续上涨天数 → 振幅（小优先） → 成交额

## 配置说明

`config.js` 主要配置项：

```javascript
TOP_N: 5             // 输出股票数量
MIN_SCORE: 5.0        // 最低推送分数阈值（动态权重体系）
MIN_PRICE: 3          // 最低价格
MAX_PRICE: 40         // 最高价格
MIN_PCT: 4            // 最低涨幅
MAX_PCT: 8            // 最高涨幅（动态调整）
MIN_TURNOVER: 3       // 最低换手率
KLINE_DAYS: 5         // K线天数（趋势分析）
MAX_AMPLITUDE: 12     // 最大振幅阈值
MAX_PULLBACK: 5       // 最大回撤阈值
PUSHPLUS_TOKEN: ""    // PushPlus推送Token
PUSHPLUS_TOPIC: ""    // 推送群组
ENABLE_PUSH: true     // 是否启用推送
```

## 定时任务

使用 node-cron 实现定时调度：
- Cron表达式: `00 48 14 * * 1-5`
- 时区: Asia/Shanghai
- 执行时间: 周一至周五 14:48

## 微信推送

使用 PushPlus 服务推送结果到微信：
- 支持白天/夜间模式自动适配
- 卡片式排版，Top3 高亮显示
- 市场情绪卡片（涨停/跌停家数、涨跌比）
- 股票标签：连涨、稳、缩量、板块热度
- 诱多信号警告
- 当前策略提示（追涨优先/均衡配置/保守为主）

**标签说明**
- 🟢 **N连涨**: 连续上涨 ≥ 3天
- 🟡 **稳**: 振幅 < 6%
- 🔵 **缩量**: 量比 < 1.5
- 🔴 **板块+N%**: 所属板块涨幅 > 3%

获取Token：关注微信公众号"PushPlus推送"，回复"token"获取。

## 数据来源

| 数据 | API | 说明 |
|------|-----|------|
| 实时行情 | `qt.gtimg.cn/q=` | 涨幅、价格、换手率、量比、振幅等 |
| K线数据 | `web.ifzq.gtimg.cn/appstock/app/fqkline/get` | 历史K线（前复权） |
| 板块数据（主） | `push2.eastmoney.com/api/qt/clist/get` | 东方财富行业板块涨幅 |
| 板块数据（降级） | `q.10jqka.com.cn/thshy/` | 同花顺行业板块涨幅 |

## 板块数据降级机制

1. 优先使用东方财富API获取行业板块涨幅
2. 东方财富超时/失败时，自动降级到同花顺
3. 使用热门板块（前5名）平均涨幅作为市场热度参考
4. 缓存5分钟，避免频繁请求

## 开发规范

1. **模块化设计**: 每个因子独立文件，便于维护和调整
2. **ES Modules**: 使用 import/export 语法
3. **异步处理**: 使用 async/await
4. **编码处理**: 腾讯API返回GBK编码，需用 TextDecoder 转换
5. **请求控制**: 批量请求间添加延迟，避免触发限流
6. **浮点精度**: 评分计算使用 toFixed(2) 避免精度问题
7. **降级容错**: 板块数据支持主备切换

## 注意事项

1. 数据来源于公开API，仅供学习研究
2. 选股结果仅供参考，不构成投资建议
3. 定时任务建议在交易时间内执行（14:48左右）
4. 动态权重会根据市场情绪自动调整策略风格
5. 换手率过高的股票会被自动过滤
6. 诱多识别会扣分，冲高回落等形态会被标记
